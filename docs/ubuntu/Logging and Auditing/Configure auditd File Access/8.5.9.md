# Ensure audit tools owner is configured

The necessity of this configuration is to protect the integrity of the system's auditing capabilities. The audit tools are critical for monitoring and recording security-relevant events. If an unauthorized user gains ownership of these files, they could modify, replace, or disable them, effectively blinding the system to malicious activity and preventing forensic analysis. Misconfiguration creates a severe risk of undetected system compromise. The audit script checks the ownership of the core audit binaries, while the remediation script uses the `chown` command to reset the owner to `root`. All recommended states (Basic, Moderate, and Strict) mandate 'root' as the owner. This is an absolute requirement, as there is no acceptable scenario where a non-root user should own these foundational security tools. This configuration adheres to the principle of least privilege and is a universal best practice for securing system logging and auditing infrastructure.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `['root']` |
| **Moderate** | `['root']` |
| **Strict** | `['root']` |
| **Type** | `ENUM` |

## Audit Script Working

Bash script that identifies the owner of the audit tool binaries (`auditctl`, `aureport`, `ausearch`, `autrace`, `auditd`) located in the `/sbin` directory. The script should verify that all these files have the same owner. If they do, it should output the owner's name (e.g., 'root'). If the ownership is inconsistent, it should output the string 'Mixed'. The output must consist solely of the resulting string.

## Remediation Script Working

Bash script that accepts a single positional command-line argument, which is the desired username for the owner (e.g., 'root'). The script must use this argument to set the ownership for all audit tool binaries (`auditctl`, `aureport`, `ausearch`, `autrace`, `auditd`) in the `/sbin` directory using the `chown` command. The script must return `true` if the operation is successful and `false` otherwise.
