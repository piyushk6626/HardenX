# Ensure latest version of pam is installed

Necessity: Pluggable Authentication Modules (PAM) are a critical component of the Linux security infrastructure, managing all aspects of user authentication, authorization, and session management. Vulnerabilities in PAM can have severe consequences, potentially allowing attackers to bypass authentication controls, escalate privileges, or gain unauthorized access to the system. Ensuring the latest version of the 'pam' package is installed is essential for applying security patches that fix known vulnerabilities, thereby protecting the system's core authentication mechanisms. Failure to keep PAM updated leaves the system exposed to significant security risks.

Auditing and Remediation: The audit script verifies the status of the 'pam' package by querying the system's package manager. It compares the installed version with the latest version available in the official repositories to determine if an update is required. The remediation script automates the security update process by using the package manager ('yum' or 'dnf') to install the 'pam' package if it's missing or upgrade it to the latest version, ensuring all security patches are applied.

Recommended State Rationale: 
Basic, Moderate, and Strict: For a foundational security package like PAM, there is no acceptable level of compromise. The only secure state is having the latest, fully patched version installed. Any outdated version may contain known vulnerabilities that could be exploited. Therefore, all recommended security levels—Basic, Moderate, and Strict—mandate the state 'Installed and Up-to-date'. This uniform recommendation underscores the non-negotiable importance of maintaining the integrity of the system's authentication framework.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `['Installed and Up-to-date']` |
| **Moderate** | `['Installed and Up-to-date']` |
| **Strict** | `['Installed and Up-to-date']` |
| **Type** | `ENUM` |

## Audit Script Working

Bash script that checks if the 'pam' package is installed and is the latest version available in the configured repositories. The script should output 'Installed and Up-to-date' if the package is installed and no updates are available, 'Update Available' if an update is found, and 'Not Installed' if the package is not present. The output must be only one of these strings, with no additional text or formatting.

## Remediation Script Working

Bash script that accepts one argument representing the desired state. If the argument is 'Installed and Up-to-date', the script should use the system's package manager (e.g., yum or dnf) to install or update the 'pam' package to the latest available version. The script must return 'true' if the operation completes successfully and 'false' if it fails.
