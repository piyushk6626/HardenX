# Ensure audit log files mode is configured

The necessity of configuring the audit log file mode lies in protecting the integrity and confidentiality of the audit trail. These logs contain sensitive information about system events, including user actions, authentication attempts, and potential security violations. If unauthorized users can read these logs, they could gain insight into system vulnerabilities or user activities. If they can modify or delete the logs, an attacker could cover their tracks, completely undermining the purpose of the audit system. The risk of misconfiguration is a critical security gap, as it can lead to undetected breaches and failed compliance audits.

The audit script checks the current permission mode set for new audit log files by reading the `log_file` parameter within the `/etc/audit/auditd.conf` configuration file. The remediation script directly modifies this parameter to enforce the desired permission settings, ensuring that any new log files created by the audit daemon adhere to the security policy.

The recommended states are justified as follows:
- **Basic (`0640`):** This state ensures that only the root user can write to the logs, while both root and members of the root group can read them. This is a solid baseline for preventing non-privileged users from accessing sensitive log data.
- **Moderate & Strict (`0600`):** This is the most secure and commonly recommended configuration, aligning with CIS Benchmarks. By setting the mode to `0600`, read and write access is restricted exclusively to the root user. This enforces the principle of least privilege, preventing even other privileged groups from directly accessing the raw log files. This enhances log integrity and confidentiality by forcing all access to audit data to go through proper administrative channels and tools like `ausearch` and `aureport`.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `['0640']` |
| **Moderate** | `['0600']` |
| **Strict** | `['0600']` |
| **Type** | `ENUM` |

## Audit Script Working

Bash script that queries the `/etc/audit/auditd.conf` file for the `log_file` parameter. The script must parse this file and return only the numeric mode value (e.g., '0600').

## Remediation Script Working

Bash script that accepts a single numeric file mode (e.g., '0600') as a positional command-line argument. The script must modify the `/etc/audit/auditd.conf` file, setting the `log_file` parameter to the provided value. The script must return 'true' on successful modification and 'false' on failure.
