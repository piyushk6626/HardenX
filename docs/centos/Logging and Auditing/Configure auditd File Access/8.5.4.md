# Ensure the audit log file directory mode is configured

The necessity of this setting is to protect the integrity and confidentiality of audit logs. Audit logs contain sensitive information about system events, user actions, and potential security breaches. If the log directory has insecure permissions, an unauthorized user or attacker could read the logs to gather intelligence, or worse, modify or delete them to cover their tracks, rendering forensic analysis impossible. Misconfiguration creates a significant gap in the system's security posture.

The audit script verifies the current file permissions of the audit log directory, which is typically '/var/log/audit'. The remediation script uses the 'chmod' utility to set these permissions to a more secure value.

The recommended states are justified as follows:
- **Basic ('0750'):** This state (rwxr-x---) allows the owner (root) full control and the group (typically 'root') read and execute permissions, while denying any access to other users. This is a strong security baseline recommended by benchmarks like CIS, ensuring logs are protected from general users but accessible to privileged accounts and processes.
- **Moderate & Strict ('0700'):** This state (rwx------) is more restrictive, providing read, write, and execute permissions only to the owner (root). It completely isolates the audit logs from all other users on the system, including those in the 'root' group. This adheres to the principle of least privilege in its strictest form and is the recommended configuration for high-security environments where log access must be maximally controlled.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `['0750']` |
| **Moderate** | `['0700']` |
| **Strict** | `['0700']` |
| **Type** | `ENUM` |

## Audit Script Working

Bash script that determines the directory path of the audit logs by parsing the 'log_file' parameter from '/etc/audit/auditd.conf'. The script should then use the 'stat -c "%a"' command to find the octal permission mode of that directory. The script must output only the numerical permission mode (e.g., '750') and nothing else.

## Remediation Script Working

Bash script that accepts a single positional argument representing the desired octal permission mode (e.g., '750'). The script must first identify the audit log directory from the 'log_file' setting in '/etc/audit/auditd.conf' and then use the 'chmod' command to apply the specified mode to that directory. The script must output 'true' upon successful execution or 'false' if an error occurs.
