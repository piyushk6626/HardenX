# Ensure 'Windows Firewall: Private: Logging: Name' is set to '%SystemRoot%\System32\logfiles\firewall\privatefw.log'

Necessity: Enabling and correctly configuring firewall logging is critical for security monitoring, incident response, and forensic analysis. The log files record allowed and blocked connections, which helps in identifying unauthorized access attempts, malware activity, and network reconnaissance. If the log file path is misconfigured or points to a non-existent location, these crucial security events will not be recorded, creating a significant blind spot in the organization's security posture. For the Private profile, which applies to trusted networks, logging is essential for detecting insider threats or compromised devices moving laterally within the network. Auditing and Remediation: The audit script checks the 'LogFileName' property for the Private network profile to ensure logs are being saved to the correct location. The remediation script configures this path, directing the firewall to write its logs to the specified file, thereby ensuring the integrity of the security monitoring data. Recommended State Rationale: The recommended state for all levels (Basic, Moderate, and Strict) is '%SystemRoot%\System32\logfiles\firewall\privatefw.log'. This is the default and industry-accepted location for Windows Firewall logs. Using a standardized path simplifies log aggregation and analysis by Security Information and Event Management (SIEM) systems and other monitoring tools. There is no inherent security advantage in using a non-standard path; in fact, it often complicates management and can lead to logs being overlooked. Therefore, the single, correct configuration is recommended for all hardening levels.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `['%SystemRoot%\System32\logfiles\firewall\privatefw.log']` |
| **Moderate** | `['%SystemRoot%\System32\logfiles\firewall\privatefw.log']` |
| **Strict** | `['%SystemRoot%\System32\logfiles\firewall\privatefw.log']` |
| **Type** | `ENUM` |

## Audit Script Working

PowerShell script that retrieves the log file name for the Windows Defender Firewall's Private profile. The script must use the 'Get-NetFirewallProfile' cmdlet, select the 'LogFileName' property, and expand any environment variables in the path. The output must be only the full file path string, with no additional text or labels.

## Remediation Script Working

PowerShell script that sets the Windows Defender Firewall log file name for the Private profile. The script must accept the desired file path as a single, positional command-line argument. It should use the 'Set-NetFirewallProfile' cmdlet to apply the setting. The script must return '$true' if the operation is successful and '$false' if it fails.
