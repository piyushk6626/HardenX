# Ensure 'Windows Firewall: Private: Logging: Size limit (KB)' is set to '16,384 KB or greater'

Necessity: Maintaining adequate firewall log sizes is critical for security monitoring, incident response, and forensic analysis. These logs record successful and dropped connections, providing a detailed history of network traffic that can be used to detect reconnaissance activities, unauthorized access attempts, and malware communications. If the log file size is too small, critical event data can be quickly overwritten and lost, severely hampering the ability to investigate and respond to a security incident. Auditing and Remediation: The audit script verifies the current log size limit by querying the 'LogFileSizeKilobytes' property of the private firewall profile. The remediation script modifies this same property, allowing an administrator to enforce a specific log size policy. Recommended State Rationale: The recommended states are designed to provide increasing levels of log retention. 'Basic' ([16384, 16384]) aligns with common security benchmarks (e.g., CIS) and ensures a minimal amount of recent data is captured. 'Moderate' ([32768, 32768]) doubles the retention capacity, offering a better balance for environments that require visibility over a few days. 'Strict' ([65536, 'upper bound']) is intended for high-security environments where comprehensive, long-term log retention is necessary for deep forensic analysis and compliance with stringent regulatory standards.

## Recommended State Configuration

| Security Level | Value |
|----------------|-------|
| **Basic** | `[16384, 'upper bound']` |
| **Moderate** | `[32768, 'upper bound']` |
| **Strict** | `[65536, 'upper bound']` |
| **Type** | `RANGE` |

## Audit Script Working

PowerShell script that retrieves the value for 'Windows Firewall: Private: Logging: Size limit (KB)'. The script must use the 'Get-NetFirewallProfile' cmdlet to query the 'Private' profile and select the 'LogFileSizeKilobytes' property. The output must be the numerical value only, with no additional text or formatting.

## Remediation Script Working

PowerShell script that sets the 'Windows Firewall: Private: Logging: Size limit (KB)'. The script must accept the desired size in kilobytes as a single, positional command-line argument. It should use the 'Set-NetFirewallProfile' cmdlet to apply the setting to the 'Private' profile. The script must return '$true' on success and '$false' on failure.
